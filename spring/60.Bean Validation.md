
## Validation 분산
- 데이터 유효성 검사 로직은 따로 관리하지 않으면 애플리케이션 전체로 분산된다.
- 이에 따라 코드 중복이 심해지고
- 비즈니스 로직과 뒤섞여 점점 복잡해진다.
## 그래서, Bean Validation
- Java에서 2009부터 유효성 검사 프레임워크를 제공하고 있고 이를 바탕으로 제약을 도메인 모델에 정의할 수 있게 됐다.
## Spring과는?
- Spring은 `spring-boot-starter-validation`을 제공하고 있다.
- 또한 내부적으로 Bean Validation을 래핑하여 유효성 검사를 사용하고 있다.

## Collection
- 컨테이너 요소, Collection 요소에 대해서 유효성 감사를 할 수 있게 됐으며(Bean Validation 2.0부터) 이를 통해서 유연하게 정합성을 지킬 수 있다.
```java
public class DeleteContacts {
    @Min(1)
    private Collection<@Length(max = 64) @NotBlank String> uids;
}
```

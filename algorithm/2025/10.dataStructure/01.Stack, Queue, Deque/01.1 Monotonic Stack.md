## 단조 스택

### 정의
- 스택 내부 원소가 단조 증가, 단조 감소 순서를 유지하도록 관리하는 스택
- 새 원소를 push 할 때, 단조성을 깨는 기존 원소를 pop하고 처리한 뒤 push


| 순회 방향 | 스택 단조성                       | 용도                              |
| :---: | ---------------------------- | ------------------------------- |
| 왼→오감소 | (top이 작음)각 원소의 **오른쪽 첫 큰 값** | (NGE: Next Greater Element)     |
| 왼→오증가 | (top이 큼)각 원소의 **오른쪽 첫 작은 값** | (NSE: Next Smaller Element)     |
| 오→왼감소 | (top이 작음)각 원소의 **왼쪽 첫 큰 값**  | (PGE: Previous Greater Element) |
| 오→왼증가 | (top이 큼)각 원소의 **왼쪽 첫 작은 값**  | (PSE: Previous Smaller Element) |

### O(n)
- 각 인덱스는 **정확히 한 번 push**
- 각 인덱스는 **최대 한 번 pop**
- 총 연산: 2N → O(N)
- for 안에 while이 있어도 **amortized** O(N)

> - 각 원소에 대해, 자기보다 오른쪽(또는 왼쪽)에서 **처음으로 자기보다 큰/작은** 원소를 찾아라" → 단조 스택
> - 스택에는 인덱스를 저장 (값은 배열에서 조회)
> - 스택의 값은 **단조 감소(또는 증가)** 상태를 유지
> - 새 값이 들어올 때, 단조성을 깨는 동안 pop → pop된 인덱스의 답은 새 값
> - 끝까지 안 깨진 인덱스는 답 없음 (-1)

> - 주식 가격 (stock prices) — 단조 스택의 변형
> - 큰 수 만들기 — 단조 스택의 변형
> - 히스토그램에서 가장 큰 직사각형 — 단조 스택의 정수

```kotlin
/**
 * # 뒤에 있는 큰 수 찾기
 * ## 문제 설명
 * 
 * 정수로 이루어진 배열 `numbers`가 있습니다. 배열 의 각 원소들에 대해 자신보다 뒤에 있는 숫자 중에 * 서 자신보다 크면서 가장 가까이 있는 수를 뒷 큰수라고 합니다.  
 * 정수 배열 `numbers`가 매개변수로 주어질 때, 모든 원소에 대한 뒷 큰수들을 차례로 담은 배열을  * return 하도록 solution 함수를 완성해주세요. 단, 뒷 큰수가 존재하지 않는 원소는 -1을 담습니다.
 * 
 * ---
 * 
 * ##### 제한사항
 * 
 * - 4 ≤ `numbers`의 길이 ≤ 1,000,000
 * - 1 ≤ `numbers[i]` ≤ 1,000,000
 * 
 * ---
 * 
 * ##### 입출력 예
 * 
 * |numbers|result|
 * |---|---|
 * |[2, 3, 3, 5]|[3, 5, 5, -1]|
 * |[9, 1, 5, 3, 6, 2]|[-1, 5, 6, 6, -1, -1]|
 */

fun calculate(numbers: IntArray): IntArray {  
    val arrayDeque = ArrayDeque<Int>()  
    val copyOf = numbers.copyOf()  
  
  
    for( i in 0 until numbers.size) {  
        while(arrayDeque.isNotEmpty() && numbers[arrayDeque.last()] < numbers[i]) {  
            val index = arrayDeque.removeLast()  
            copyOf[index] = copyOf[i]  
        }  
  
        arrayDeque.addLast(i)  
    }  
  
    while(arrayDeque.isNotEmpty()) {  
        val index = arrayDeque.removeLast()  
        copyOf[index] = -1  
    }  
    return copyOf  
}  
  
  
@Test  
fun solution1() {  
    val numbers = intArrayOf(2, 3, 3, 5)  
    val result = intArrayOf(3, 5, 5, -1)  
  
    assertArrayEquals(result, calculate(numbers))  
}  
  
  
@Test  
fun solution2() {  
    val numbers = intArrayOf(9, 1, 5, 3, 6, 2)  
    val result = intArrayOf(-1, 5, 6, 6, -1, -1)  
  
    assertArrayEquals(result, calculate(numbers))  
}



/**
  * # 주식_가격
  * ## 문제 설명
  *초 단위로 기록된 주식가격이 담긴 배열 prices가 매개변수로 주어질 때, 가격이 떨어지지 않은 기간  *은 몇 초인지를 return 하도록 solution 함수를 완성하세요.
  *
  *##### 제한사항
  *
  *- prices의 각 가격은 1 이상 10,000 이하인 자연수입니다.
  *- prices의 길이는 2 이상 100,000 이하입니다.
  *
  *##### 입출력 예
  *
  *|prices|return|
  *|---|---|
  *|[1, 2, 3, 2, 3]|[4, 3, 1, 1, 0]|
  */

  int[] calculate(int[] prices) {  
    ArrayDeque<Integer> arrayDeque = new ArrayDeque<>();  
    int[] copy = new int[prices.length];  
  
    for(int i = 0; i < prices.length; i++) {  
        while(!arrayDeque.isEmpty() && prices[arrayDeque.getLast()] > prices[i]) {  
            int index = arrayDeque.removeLast();  
            copy[index] = i - index;  
        }  
  
        arrayDeque.addLast(i);  
    }  
  
    int max = prices.length - 1;  
  
    while(!arrayDeque.isEmpty()) {  
        int index = arrayDeque.removeLast();  
        copy[index] = max - index;  
    }  
  
  
    return copy;  
}


```

> 1. **메인 루프와 잔여 처리는 상호 배타적**이다.
>     - 메인 루프 pop → "떨어진 사건이 발생함" → `i - index`
>     - 잔여 처리 → "끝까지 떨어진 사건 없음" → `(n-1) - index`
> 2. **잔여 처리에는 보정이 필요 없다.**
>     - 잔여에 도달했다는 것 자체가 "이후로 단조성을 거스르는 값이 없었다"는 증거
>     - 따라서 거리 계산은 단순히 배열 끝까지의 거리
> 3. **결과 배열의 인덱스를 헷갈리지 말 것.**
    - 디버깅 시 "이 값이 인덱스 몇의 답인지" 명확히 매핑해서 검증
